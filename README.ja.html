このアーカイブは「プログラミングScala」のサンプルコードです。

サンプルコードは、Scala2.8用とScala2.7用に分かれています。使用するScalaに対応したサンプルコードをダウンロードしてください。

Scala2.8向けのサンプルコードは、原書のサンプルコードをScala2.8で動作するように変更し、日本語版付録のサンプルコードを追加したものです。Scala 2.8.0-finalで動作確認しています。

また、Windows環境でも動作するScala2.8用のsakeを含んでいます。Scala2.7向けのソースコードは、原書のサンプルコードにWindows環境でも動作するScala2.7用のsakeを含んでいます。Scala 2.7.7-finalで動作確認しています。

サンプルコードのビルド・実行について

サンプルコードをビルド・実行するためには、

  • Scala 2.7.5(2.7.7推奨)以降、もしくはScala 2.8.0(2.8.0推奨)以降
  • Java 1.5以降
が必要です。必要に応じてインストールしてください。

code-example以下のソースコードは、sakeでビルド・実行することができます。code-example以下のビルドしたい章のディレクトリに移動し、Unix環境であれば 

../bin/sake
を、Windows環境であれば 
 ..\bin\sake.bat
を実行してください。各ディレクトリのサンプルコードをビルド・実行します。

コードの詳細なビルド・実行方法については、code-example/README.htmlを参照してください。15章および16章の実行方法については、本文を参照してください。

サンプルコードの注意点

sakeについて

サンプルコードをビルド・実行するためにsakeというツールを使用しています。sakeについては、https://github.com/deanwampler/SakeScalaBuildToolまたはcode-examples/README.htmlのsakeの部分を参照してください。

翻訳時点でsakeはWindowsに対応していなかったので、翻訳者でWindowsでも動作するように修正しましたが、まだ不具合があります。

  • Windows環境でsakeが止まることがある

    • Windows環境では、javaのデーモンプロセスと通信する際にsakeの動作が止まることがあります。この場合バッチファイルをCtrl-Cで終了し、再度sake.batを実行すれば問題なく動作します。また、回避策として、sakeを実行する前にfsc.batを実行しておくという方法もあります。引数をつけずにfsc.batを実行するとコマンドラインヘルプが表示されますが問題ありません。fsc.batを実行することでjavaのプロセスがバックグラウンドで起動されます。このjavaプロセスを立ち上げておけば、sakeは問題なく動作します。Unix環境やOSXではこの問題が発生することはありません。

Windowsに対応したsakeのソースコードは、https://github.com/kazumi007/SakeScalaBuildTool/で公開しています。

「9.4.3 イベント」のサンプルコード

「9.4.3 イベント」のサンプルコード(code-examples/Concurrency/smtpd)は翻訳時点でNagattiがScala 2.8に対応していなかったため、Scala 2.8でのテストを行なっていません。
